# Open Questions

## 最想搞清楚的问题

玩家看到实时指标后，是否会为了降低碳排和改善通风，放弃一部分便宜但高碳的煤电选择？

## 研究问题

1. 1000 万预算是否会让玩家感到真实的资源约束？
2. 三个指标（平均温度、平均风速、碳排放）是否足够让玩家理解取舍？
3. “风口放煤电会降低通风”这个规则，玩家能否通过游戏反馈自己发现？
4. 2 题问卷是否足够收集初步反馈，还是需要增加开放题？
5. JSON 导出的 layout + metrics 是否足够支持后续课堂分析？

## 需要继续查的方向

- 城市热岛和制冷能耗之间的量化关系。
- 通风廊道规划中常用的可解释指标，例如建筑高度、粗糙度、天空可视因子。
- 类似严肃游戏如何设计问卷来衡量“是否学到了东西”。
- 如果下周接入 Supabase，哪些字段应该云端保存，哪些字段只本地导出。

## 已收敛的 MVP 决策

- 核心交互：选设施、点网格放置、立即观察指标变化。
- 保留设施：煤电、风电、地源热泵。
- 保留指标：平均温度、平均风速、碳排放。
- 反馈方式：2 题问卷 + JSON 下载。
- 暂时砍掉：热力图、通风箭头、AI 顾问、CSV、云端数据库、多关卡。
